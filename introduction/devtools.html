Analyze a performance recording

After you record runtime performance or record load performance, the Performance panel provides a lot of data for analyzing the performance of what just happened.

Wow, that's an overwhelming amount of data. How can I?

Select a portion of a recording

- Overview Section

  • FPS

The FPS chart

  • Green bar - the higher the better
  • A red bar - warning, slow!!
60FPS is awesome!

The CPU chart

The colors in the CPU chart correspond to the colors in the Summary tab.
Full of color means that the CPU was maxed out. That's a cue!

Frames

View main thread activity

  • Random colors
  • Readable(really?)
  • Visualization of CPU stack tracking
  • JavaScript、Style calculations、Layout and Painting

Flame Chart

  • The x-axis represents the recording over time
  • Each bar represents an event - A wider bar means that event took longer
  • The y-axis represents the call stack
  • The events on top cause the events below it
  • Dash lines
    • Blue - DomContentLoaded
    • Red - Load
Notes: Whenever you see a red triangle, it's a warning that there may be an issue related to this event.
